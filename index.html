CSS选择器 (1)
更新于 2019-12-05

ID选择器

CSS：#name { }
HTML：id="name"


注：

1.在一个页面中，ID之前是唯一的，只能出现一次。
2.命名规范：字母，下划线，中划线，数字（命名第一位不能是数字）
3.命名方式：驼峰式，下划线式，短线式。

例如：

searchButton（小驼峰）
SearchButton（大驼峰）
search-small-button（短线式）
search_small_button（下划线式）

CLASS选择器

. name{ }
class="name"


注：

  • class选择器可以复用。

  • 可以添加多个class样式。

  • 多个样式的时候，样式的优先级是根据CSS决定，而不是class属性中的顺序决定。

  • 标签+类的写法。
    这是给这个盒子同时添加两个样式的时候也可以实现。

    层次选择器

后代选择器

M  N { }

在指定的条件下找到指定的需求，如下图给ul标签下的li加上指定的样式：

父子

M > N { }

只给某个元素的子元素添加样式，其后代的后代不能添加。

兄弟

M ~ N { }

给当前M下的所有兄弟N标签加上样式


这个给div下面的h2标签就加上了背景色，而它上面的h2就没有加上。

相邻

M + N { }

作用的是当前M相邻的N标签（挨在它下面的一个）


这个就给div下面的h2加上了背景。

文档流

文档流是文档中可显示对象在排列时所占用的位置

（块标签在文档流中的默认排列方式是从上到下，内联标签在文档流中的默认排列方式是从左到右）

float特性

加浮动的元素，会脱离文档流，会延迟父容器靠左或靠右排列，如果之前已经有浮动的元素，会挨着浮动的元素进行排列。

float取值

right
left
none（默认值：不具备 浮动）

那如果我给这个box2也添加一个左浮动，那么这个红色的方块也会沿着父容器靠左，但是左边已经有黄色方块了，这时候它就会挨着黄色的方块靠右进行排列。

float的注意事项

  • 只会影响后面的元素
  • 内容默认提升半层
  • 默认宽根据内容决定
  • 换行排列
  • 主要给块元素添加来实现布局，也可以给内联元素添加（但是很少用到）

就比如下面这张图片，我们是给红色的方块加了一个左浮动，让它浮在了上面，然后再给蓝色方块里加入文字，发现文字并没有被红色方块所覆盖，因为这个文字的层级要高于蓝色方块，所以它被挡到了外面。

这样的一个现象可以做一些图文混排的效果，红色区域可以做一个图片，然后文字区域可以做一些图片的描述

这个就是当浮动的元素在父容器中排列不下的情况下，它就会自动折行，会排到下面。

由于浮动会影响到后面元素的布局，而有的时候我们不希望下面的元素受到影响，所以我们要考虑清除浮动：

如何清除浮动？

上下排列：clear属性，表示清除浮动的 left、right、both
嵌套排列：
固定宽高：不推荐，不能把高度固定死，不适合做自适应的效果。
父元素浮动：不推荐，因为父容器浮动也会影响到后面的元素
overflow：hidden（BFC规范），如果有子元素想溢出，那么会受到影响。
display：inline-block（BFC规范），不推荐，父容器会影响到后面的元素。
设置空标签：不推荐，会多添加一个标签。
after伪类：推荐，是空标签的加强版，目前各大公司的做法。
（clear属性指挥操控块标签，对内联标签不起作用）
